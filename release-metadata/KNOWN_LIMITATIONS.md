# Known Limitations — v0.0.1

- Hobby/research-grade release; not industrially or functionally safety certified.
- Teensy 4.0 remains the primary low-level controller; STM32 migration is planned.
- Electronics, power distribution, charging, emergency-stop, watchdog, fuse, hot-swap, and electromagnetic-compatibility architecture require further industrialization.
- Navigation, visual odometry, auto-docking, firmware, and UI require additional integration and reliability testing.
- External operating-system, ROS, Docker, Python, JavaScript, and firmware-toolchain dependencies are not fully bundled for disconnected installation.
- Documentation completeness varies by component; empty placeholder directories may exist.
- Users must perform their own risk assessment before operating physical hardware.
